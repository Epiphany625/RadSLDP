# Reason-for-Visit Categorization Pipeline

An end-to-end pipeline for normalizing, splitting, clustering, and categorizing radiology reason-for-visit text from three chest X-ray datasets: **MIMIC-CXR**, **CheXpert+**, and **RexGradient**.

## Input Data: Where to Place It & Required Fields

Create an `original_data/` directory **in the working directory you run the scripts from** (i.e. a sibling of this `categorization/` folder) and put the 5 source JSON files in it:

```
workdir/
├── original_data/                  # <-- place the source datasets here
│   ├── chat_train_MIMIC_CXR_all_gpt4extract_rulebased_v1.json
│   ├── chat_dev_MIMIC_CXR_all_gpt4extract_rulebased_v1.json
│   ├── chat_test_MIMIC_CXR_all_gpt4extract_rulebased_v1.json
│   ├── df_chexpert_plus_240401.json     # CheXpert+ source CSV converted to JSON
│   └── rexgradient_train.json           # produced by datasets/ReXGradient/processing/process.py
└── categorization/                 # this directory (the scripts)
```

Each source file is a JSON **list of entry objects**. Only the following fields are **essential** (any other fields are ignored):

| Dataset | File | ID field (required) | Reason field(s) (required) |
|---------|------|---------------------|----------------------------|
| MIMIC-CXR | `chat_{train,dev,test}_MIMIC_CXR_*.json` | `id` | `reason` |
| CheXpert+ | `df_chexpert_plus_240401.json` | `path_to_image` | `section_clinical_history` and/or `section_history` (concatenated) |
| RexGradient | `rexgradient_train.json` | `id` | `indication` |

Notes:
- An entry with an empty/missing reason field is skipped (for augmentation it still gets empty category fields).
- Internal keys are formed as `mimic:{id}`, `chexpert:{path_to_image}`, `rex:{id}` — so the ID field must be unique within each dataset.
- The default file/dir names above can be overridden via the `DATA_DIR` environment variable (see [Environment Variables](#environment-variables)).
- All other files under `original_data/` and `sweep_result_clean_split/` are **generated** by the pipeline — see the full [Directory Structure](#directory-structure) below.

## Quick path: download the pre-built taxonomy

If you only want to **use** the taxonomy (i.e., reproduce the paper's training
runs) and do not need to regenerate it, skip Steps 1-10 entirely and fetch the
published artifact from Hugging Face:

```bash
python3 categorization/download_taxonomy.py
# → categorization/RadSLDP_nvcc_taxonomy.json
```

The full pipeline below is only needed if you want to **rebuild** the taxonomy
from raw clinical-indication text (e.g., to extend it to a new dataset).

## Source Dataset Formats

Each dataset has a different schema. The pipeline extracts reason text using these fields:

| Dataset | File | ID field | Reason field(s) |
|---------|------|----------|-----------------|
| MIMIC-CXR | `chat_{train,dev,test}_MIMIC_CXR_*.json` | `entry["id"]` | `entry["reason"]` |
| CheXpert+ | `df_chexpert_plus_240401.json` | `entry["path_to_image"]` | `entry["section_clinical_history"]` + `entry["section_history"]` |
| RexGradient | `rexgradient_train.json` | `entry["id"]` | `entry["indication"]` |

Internal keys are formatted as `mimic:{id}`, `chexpert:{path_to_image}`, `rex:{id}`.

## Step-by-Step Execution

All commands assume you `cd` into the working directory that contains `original_data/`.

### Step 1: Build Reason Index

Normalizes reason text (expands abbreviations, splits demographics), splits by `;.,`, deduplicates, and assigns indices sorted by word count (longest first).

```bash
python categorization/build_reason_index.py
```

**Inputs:** 3 source dataset JSONs in `original_data/`
**Outputs:**
- `original_data/reason_index.json` — list where `list[idx] = reason_text` (idx 0 = longest)
- `original_data/id_to_reasons.json` — `{dataset:id -> [reason_idx, ...]}`
- `original_data/reason_to_ids.json` — `{reason_idx -> [{dataset, id}, ...]}`

### Step 2: Split Compound Reasons (LLM)

Sends reasons with word count >= 6 to GPT-4.1-mini to split compound phrases (e.g. "chest pain and shortness of breath" → ["chest pain", "shortness of breath"]).

```bash
python categorization/split_compound_reasons.py
```

**Inputs:** `original_data/reason_index.json`
**Outputs:**
- `original_data/compound_split_map.json` — `{reason_idx: [split_parts]}`
- `original_data/compound_split_log.jsonl` — batch log for resume

Resumable: re-run to continue from where it left off.

### Step 3: Encode and Cluster

Builds atomic parts (short reasons as-is + compound-split parts for longer ones), deduplicates, encodes with SentenceTransformer (`pritamdeka/S-PubMedBert-MS-MARCO`), and runs a parameter sweep over UMAP + HDBSCAN.

```bash
python categorization/encode_and_clustering_clean_split.py
```

**Inputs:** `original_data/reason_index.json`, `original_data/compound_split_map.json`
**Outputs in `sweep_result_clean_split/`:**
- `part_to_reason_idxs.json` — `{part_text -> [reason_idxs]}`
- `embeddings.npy` — (N_parts, 768) array
- One subdirectory per parameter combo (e.g. `nn15_nc16_mcs100_ms50/`) with `clusters.json`, `metrics.json`

**Parameter sweep:** 81 combinations of:
- UMAP: n_neighbors ∈ {10, 15, 30}, n_components ∈ {5, 8, 16}
- HDBSCAN: min_cluster_size ∈ {30, 50, 100}, min_samples ∈ {10, 25, 50}

Resumable: skips runs where `metrics.json` already exists.

### Step 4: Summarize Clusters (LLM)

First, create a `cluster_details.json` with representative + sample texts for each cluster. Then run the summarizer.

```bash
python categorization/summarize_clustering.py <run_dir>/cluster_details.json
```

For example:
```bash
python categorization/summarize_clustering.py \
    sweep_result_clean_split/nn15_nc16_mcs100_ms50/cluster_details.json
```

**Inputs:** `cluster_details.json` (cluster representative + up to 30 samples)
**Outputs:** `cluster_summaries.json` in the same directory

### Step 5: Assign Noise to Clusters

HDBSCAN labels many points as noise (cluster -1). This step assigns each noise point to the nearest cluster centroid via cosine similarity in the 768-dim embedding space.

Handles placeholder text: `___f` → "female", `___m` → "male" for embedding lookup (the original text is preserved).

```bash
python categorization/assign_noise_to_clusters.py
```

**Inputs:** `embeddings.npy`, `clusters.json`, `cluster_summaries.json`, `part_to_reason_idxs.json`
**Outputs:**
- `clusters_full.json` — all points assigned (noise points have `from_noise: true`)
- `noise_assignments.json` — `{part_text: {assigned_cluster, cluster_summary, similarity}}`

### Step 6: Categorize Clusters (LLM)

Iteratively categorizes cluster summaries into a taxonomy of 10–25 broad categories using GPT.

```bash
python categorization/categorize_pipeline_openai.py
```

**Inputs:** `cluster_summaries.json`
**Outputs in `categorize_result_5.2/`:**
- `category_taxonomy.json` — list of categories with their cluster members
- `categorize_log.jsonl` — per-cluster categorization decisions (includes `image_inferable`)

### Step 7: Define Final Categories (Manual)

Review `category_taxonomy.json` and create two files manually:

**`final_category.json`** — the final category definitions:
```json
{
    "Category Name": "One-sentence definition",
    ...
}
```

**`category_mapping.json`** — maps old category names to final ones:
```json
{
    "Final Category Name": {
        "definition": "...",
        "original_categories": ["Old Category A", "Old Category B"]
    },
    ...
}
```

Place both in the `categorize_result_*/` directory.

### Step 8: Generate Final Outputs

Combines all pipeline outputs into per-sample categorization.

```bash
python categorization/generate_final_outputs.py
```

**Inputs:** clusters, noise assignments, summaries, taxonomy, mapping, reason index, id_to_reasons, compound split map
**Plus (`AUGMENTED_EXTRAS`):** any already-augmented files for datasets not in the
core pipeline — by default `original_data/augmented/rexgradient_test_augmented.json`
(produced by Step 9) — whose entries are merged in under a `rex_test:` key prefix.
If that file does not exist yet, it is skipped with a warning (see ordering note below).
**Outputs:**
- `id_to_categorized_reasons.json` — detailed per-sample output:
  ```json
  {
      "mimic:12345": {
          "reasons": [{
              "reason_idx": 42,
              "reason_text": "chest pain",
              "parts": [{
                  "part_text": "chest pain",
                  "cluster_id": 7,
                  "cluster_summary": "...",
                  "final_category": "Clinical Symptoms",
                  "image_inferable": "yes",
                  "from_noise": false
              }]
          }]
      }
  }
  ```
- `id_to_final_categories.json` — flat summary: `{dataset:id -> {image_inferable, final_categories}}`

### Step 9: Augment Training Data

Adds the categorized reason parts as new attributes to each sample in all 6 source files.

```bash
python categorization/augment_training_data.py
```

For each reason part the script first tries the **pre-computed** mapping
(`id_to_categorized_reasons.json`, covering MIMIC train / CheXpert+ / Rex train).
For **unseen** parts — those not in the training clustering, e.g. most MIMIC
**dev/test** and Rex **test** reasons — it falls back to **nearest-centroid
assignment**: the part is encoded with `pritamdeka/S-PubMedBert-MS-MARCO` and
assigned to the cluster whose centroid (mean embedding) is most cosine-similar.
This needs the extra inputs `embeddings.npy` and `part_to_reason_idxs.json`.

**Inputs:** 6 original dataset files + `id_to_categorized_reasons.json`
(+ `embeddings.npy`, `part_to_reason_idxs.json`, clusters/noise/summaries/taxonomy
/mapping for the on-the-fly lookup)
**Outputs in `original_data/augmented/`:**
1. `chat_train_MIMIC_CXR_augmented.json`
2. `chat_dev_MIMIC_CXR_augmented.json`
3. `chat_test_MIMIC_CXR_augmented.json`
4. `df_chexpert_plus_augmented.json`
5. `rexgradient_train_augmented.json`
6. `rexgradient_test_augmented.json`

Each sample gets three new fields:
- `reason_parts` — full categorization detail (reason_text, parts with categories)
- `final_categories` — list of unique category names
- `has_image_inferable` — boolean

> **Step 8 ↔ 9 ordering.** Step 8's `AUGMENTED_EXTRAS` reads
> `rexgradient_test_augmented.json` (a Step 9 output), while Step 9 reads
> `id_to_categorized_reasons.json` (a Step 8 output). Resolve this iteratively:
> run **Step 8** (the Rex-test extra is skipped if absent) → run **Step 9** →
> run **Step 8 again** to fold the `rex_test:*` entries into the final
> `id_to_categorized_reasons.json` / `id_to_final_categories.json`.

### Step 10: Apply Expert Corrections (Final Clean-Up)

Of the 347 GPT-annotated cluster summaries, five were re-labeled by a certified
medical physicist (see paper §2.2). This script flips `image_inferable` from
`"no"` to `"yes"` on every reason whose `cluster_summary` falls in one of the
five corrected categories and writes the **public taxonomy file** consumed by
`training/hf_finetune.py`.

```bash
python categorization/apply_expert_corrections.py
# or override paths
INPUT=path/to/id_to_categorized_reasons.json \
OUTPUT=path/to/RadSLDP_nvcc_taxonomy.json \
  python categorization/apply_expert_corrections.py
```

**Inputs:** `id_to_categorized_reasons.json` (Step 8 output, with `rex_test:*`
entries already folded in per the iterative ordering above).
**Outputs:** `RadSLDP_nvcc_taxonomy.json` — the artifact released alongside the
paper and the file that `training/hf_finetune.py` loads as `ID_TO_CAT_JSON`.

## Notes

- **`encode_and_clustering.py`** is the legacy version that clusters full reason texts (before compound splitting). The active pipeline uses **`encode_and_clustering_clean_split.py`**.
- **`merge_categories.py`** is an optional tool for iteratively merging fine-grained categories into broader ones using GPT-4o. Not part of the core pipeline.
- Steps 2, 4, and 6 are **resumable** — re-running picks up where the previous run stopped.
- The SentenceTransformer model (`pritamdeka/S-PubMedBert-MS-MARCO`) produces 768-dim embeddings.
- Word count threshold for compound splitting is **6 words**.
